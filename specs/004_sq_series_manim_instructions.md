# 平方数の和 Σk² = n(n+1)(2n+1)/6 — Manim アニメーション制作指示書

## 概要

**テーマ：** 平方数の和の公式 `Σk² = n(n+1)(2n+1)/6` を視覚的に証明する数学アニメーション  
**対象 n：** N = 5（1² + 2² + 3² + 4² + 5²）  
**シーン数：** 9シーン  
**出力形式：** mp4（1920×1080 推奨）または 1280×720  
**使用ライブラリ：** Manim Community Edition（`manim` コマンド）

---

## カラーパレット

```python
# 背景
BG_COLOR = "#0f0f1a"

# k番目の正方形の基本色（k=1..5）
BASE = ["#d4622a", "#1a7a6a", "#2e6eb0", "#8844bb", "#b08020"]

# m番目の奇数バンドの色（m=1..5、全kで共通）
ODD_BY_M = ["#e05c20", "#2a9d60", "#2060c0", "#9933bb", "#c8a030"]

# ラベル・枠線
LABEL_GOLD  = "#c8b88a"
FORMULA_YELLOW = "#e8c87a"
FINAL_YELLOW   = "#f0d858"
GLOW_YELLOW    = "#e8c87a"
DIM_GREEN   = "#a8c8b0"
```

## 基本データ構造

```python
N = 5

# k×k 正方形の m番目 L字バンドのセル一覧
# L字バンド: max(r, c) + 1 == m となるセル(r, c)（0-indexed）
def l_band(k, m):
    return [(r, c) for r in range(k) for c in range(k)
            if max(r, c) + 1 == m]

# 全バンドリスト（k=1..N, m=1..k の順）
ALL_BANDS = [(k, m) for k in range(1, N+1) for m in range(1, k+1)]
NB = len(ALL_BANDS)  # = 15

# ODD[k-1][m-1] = m番目バンドの色
ODD = [[ODD_BY_M[m] for m in range(k)] for k in range(N)]

# 降順ソート済みバンド（高さ 2m-1 の降順）
BAND_ORDER = sorted(enumerate(ALL_BANDS), key=lambda x: -(2*x[1][1]-1))
# BAND_ORDER[si] = (orig_idx, (k, m))

# セルサイズ
CELL = 0.35  # Manim単位（画面に合わせて調整）
GAP  = 0.04
S    = CELL + GAP
SQ_GAP = 0.09  # Scene2以降の正方形間隔

# 長方形サイズ
RW = N*(N+1)//2   # = 15（列数）
RH = 2*N + 1      # = 11（行数）
```

---

## セル描画の共通ルール

各セルは以下の見た目で描画する：

- `CELL × CELL` の正方形（角なし）
- 上端に白（alpha=0.11）の薄いハイライト帯（高さ CELL の約 1/5）
- 黒（alpha=0.2）の細い枠線（線幅 0.5px 相当）
- フェードイン・アウトは `opacity` または `FadeIn` / `FadeOut` で制御

Manimでは `Square` や `Rectangle` + `VGroup` を使い、色・不透明度を設定する。

---

## 座標系の考え方

Manimのデフォルト座標系（中央原点、y軸上向き）を使用する。

**階段（staircase）の積み方：**
- k=1,2,...,N の正方形を左から順に配置
- 各正方形の底辺は共通の基準ライン `base_y` に揃える
- k番目の正方形の左端x: `ox + sum(j*S for j in range(1, k))`（隙間なし版）
- Scene2以降の隙間あり版: `ox + sum(j*S + SQ_GAP for j in range(1, k))`

**反転階段（flipped staircase）：**
- 通常階段と左右の順序は同じ（k=1が左、k=Nが右）
- ブロックが基準ライン `top_y` から「下に向かって」生える

**長方形サイズ（コピーA+B+C合体後）：**  
幅 = RW × S = 15S、高さ = RH × S = 11S

## シーン別詳細仕様

---

### Scene 1：階段の登場と Σk² 提示
**所要時間目安：3.2秒**

**内容：**
1. k=1,2,3,4,5 の正方形が**左から順番**にフェードイン（easeOut）
   - 各正方形は前の正方形が十分表示されてから次が始まる
   - タイミング例：k番目の開始は `t = (k-1)/(N+0.5)` 付近
2. 各正方形の**上に k² のラベル**を表示（正方形と同じ色、bold）
   - 例：1番目の上に「1²」、2番目の上に「2²」...「5²」
   - ラベルは正方形の水平中心、正方形上端から少し上
3. 全正方形が揃った後、**下部に式をフェードイン**（黄金色、グロー付き）：
   ```
   Σk² = 1² + 2² + 3² + 4² + 5²
   ```
   キャンバス水平中央、階段底辺より下に配置

**Manim実装メモ：**
- `LaggedStart` + `FadeIn` で各正方形のズレたフェードイン
- ラベルは `MathTex` または `Text` で `k^2` 表記
- 最後の式は `MathTex(r"\sum k^2 = 1^2 + 2^2 + 3^2 + 4^2 + 5^2")`

---

### Scene 2：L字バンドによる奇数の和への分解
**所要時間目安：6.5秒**

**内容：**
- 各 k×k 正方形を **L字バンド（m=1,2,...,k）** ごとに色分けして順に表示
- SQ_GAP の隙間を空けて配置（水平中央寄せ）
- k=1 → k=2 → ... → k=5 の順に、各kの中でバンドm=1,2,...,kが順にフェードイン
- バンドの色は `ODD_BY_M[m-1]`（m番目バンドは全kで同色）

**L字バンドの形状：**
- m番目バンド：k×k 正方形の中で `max(r,c)+1 == m` となるセル群
- m=1: 左下角の1セル
- m=2: 3セル（L字型）
- m=k: 外周L字（2k-1セル）

**タイミング：**
- k番目の正方形に割り当てた時間窓 `[（k-1)/N, k/N] × 0.88`
- その窓の中でバンドm=1..kを等分割してフェードイン

---

### Scene 3：3つのコピー A・B・C の生成
**所要時間目安：3.5秒**

**内容（左半分にA+B、右半分にCを配置）：**

1. **コピーB**（通常の階段）が左側に最初に登場（t=0〜0.2）
   - 底辺は画面垂直中央付近 `base_y`
   - 基本色（BASE）で塗る

2. **コピーA**（上下反転の階段）がコピーBから**上に分裂**（t=0.22〜0.55）
   - 左右の順序はBと同じ（k=1が左）
   - ブロックが上から「ぶら下がる」向き（下向きに生える）
   - AとBの間に1ブロック分の隙間を空けながら上昇

3. **コピーC**（奇数バンド色の階段、SQ_GAP付き）がBから**右にスライド**（t=0.35〜0.72）
   - 色は ODD（各バンドの色）
   - 右半分（画面中央より右）に移動して落ち着く

---

### Scene 4：コピーCの L字バンド分解（コ字型）
**所要時間目安：3.5秒**

**内容：**
- 左側にコピーA・B が薄く残る（alpha ≈ 0.25）
- コピーCの各 L字バンドが**個別にコ字型**（鈎括弧型）に分かれて画面下部に並ぶ
- `ALL_BANDS` の順（k=1のm=1バンドから順）にバラバラの小グループとして下部に着地
- 各バンドの到着地点は画面下部に左から等間隔（バンド内のセルはコ字型＝元の相対配置を維持）

**コ字型の意味：**  
各 L字バンドの (r,c) セル群をそのままの形状で降ろす（列として縦に並べるのではなく、元のL字配置を保持）

**タイミング：**
- バンドのインデックス順に少しずつ遅延（`delay = bi/NB × 0.5`）

---

### Scene 5：コ字型→縦I列→降順ソート→上下中央揃え
**所要時間目安：4.8秒**

**3フェーズの変形アニメーション：**

**Phase 1（t=0〜0.40）：コ字型 → I列（縦一列）**
- 各バンドのセルを **縦一列** に並べ直す（底辺揃え）
- セル数 = `2m-1`、縦に積んで I 字型に

**Phase 2（t=0.45〜0.75）：I列を降順に並べ替え**
- 高さ（セル数）の降順にソート：9,7,7,5,5,5,3,3,3,3,1,1,1,1,1（N=5の場合）
- 左から高い順に並ぶよう水平移動

**Phase 3（t=0.80〜1.0）：上下中央揃え**
- 各I列を垂直方向に中央揃え（全列の中心が同じ高さになる）
- この最終位置が Scene6 でのスライド開始位置

**注意：** 左側のコピーA・B は薄く残り続ける（alpha ≈ 0.18）

---

### Scene 6：I列が右スライドして長方形に合体
**所要時間目安：4.5秒**

**内容：**

1. **コピーA・B** が長方形のターゲット位置に現れる（t=0〜0.35、フェードイン）
   - A（反転）+ B（通常）を縦に重ねると横幅 RW × S = 15S、高さ RH × S = 11S の長方形になる
   - 長方形の右端は画面右側（`W - RW*S - 38px` 相当）

2. **破線の長方形枠**がうっすら表示される（t=0.10〜）

3. **コピーC（I列群）が右にスライドして長方形内のターゲット位置へ移動**（t=0.28〜）
   - 右端の列（高さ1のバンド）から先に動き始める（降順に遅延）
   - 各列の着地先は、A+B が作る長方形の「隙間（C の位置）」

4. **長方形ごと中央に移動**（t=0.65〜0.95）：右寄り → 画面中央

**C の着地先の計算（重要）：**  
長方形内の C 部分の位置は次のロジックで決まる：
```python
# ks番目正方形のms番目バンド → 長方形内での配置
# kd = N+1-ms  (列インデックス)
# cl = ks-ms   (列内オフセット)
# tx = ox + cum_width(kd)  + cl*S  (x座標)
# y  = bottom_y - kd*S  to  bottom_y - kd*S - (2ms-1)*S  (セル群のy座標)
```
（これが `copyCCells(ox,oy)` の計算に対応）

---

### Scene 7：長方形の完成とグロー演出
**所要時間目安：2.2秒**

**内容：**
- A+B+C が合体した完成長方形を静止表示
- 金色の輝く枠線（グロー付き）がフェードインしながら長方形を縁取る
- 枠線色：`#e8c87a`、shadowBlur 相当のグロー

---

### Scene 8：寸法ラベルと式の導出
**所要時間目安：3.2秒**

**内容（段階的に表示）：**

1. **縦方向の矢印ラベル**（t=0.05〜）：長方形右側に縦矢印と「2n+1」
2. **横方向の矢印ラベル**（t=0.32〜）：長方形下側に横矢印と「½n(n+1) = 1+2+…+n」
3. **式のフェードイン**（t=0.56〜）：長方形上部に大きく：
   ```
   3Σk² = ½n(n+1)·(2n+1)
   ```
   黄金色、グロー付き

**式の意味の解説：**
- 長方形の高さ = 2n+1
- 長方形の幅 = 1+2+…+n = n(n+1)/2
- 面積 = 3Σk²（コピーA+B+Cそれぞれが Σk² に等しいため）

**Manim実装メモ：**
- `MathTex(r"3\sum k^2 = \frac{1}{2}n(n+1)\cdot(2n+1)")`
- 矢印は `Arrow` または `DoubleArrow`

---

### Scene 9：最終公式の導出
**所要時間目安：3.5秒**

**内容：**

1. 長方形は Scene8 と同じ静止状態で維持
2. 長方形下の横矢印のみ残す（薄く）
3. 「3Σk² = …」が**フェードアウト**しながら
4. **分数形式の最終公式**がフェードイン：

```
        n(n+1)(2n+1)
Σk²  =  ────────────
               6
```

5. 長方形の右隣（または上）に式を表示、グロー演出付き

**Manimでの式：**
```python
MathTex(r"\sum k^2 = \frac{n(n+1)(2n+1)}{6}")
```

## 全シーン共通の実装方針

### イージング関数

```python
import numpy as np

def ease_out(x):
    return 1 - (1 - x)**3

def ease_in_out(x):
    if x < 0.5:
        return 4 * x**3
    else:
        return 1 - (-2*x + 2)**3 / 2
```

Manimでは `rate_func=smooth` (≈ ease_in_out) や `rate_func=rush_from` (≈ ease_out) を活用。

### フェードと透明度制御

- セルごとの個別制御が必要な場面は `VGroup` を使い `.set_opacity()` で対応
- フェードイン: `FadeIn(mob, run_time=...)` または `mob.animate.set_opacity(1)`

### シーン間の連続性

- 各シーンは **独立した `Scene` クラス** として実装し、後で `--from_scene` または結合して使う
- または1つの大きな `Scene` に全て収める（推奨：`MovingCameraScene` を継承）

### 推奨クラス構成

```python
class SqSeriesAnimation(MovingCameraScene):
    def construct(self):
        self.camera.background_color = BG_COLOR
        self.scene1()
        self.scene2()
        # ...

    def scene1(self):
        # Scene 1の実装
        pass
```
