# コッホ曲線アニメーション Manim 実装仕様書

## 概要

コッホ曲線（Koch Snowflake）のフラクタル生成過程を通して、周長が∞に発散しながら面積が有限値 8/5 A₀ に収束することを視覚的に示す教育アニメーション。

---

## グローバル設定

| 項目 | 値 |
|------|-----|
| 解像度 | 1920×1080（16:9）または 1080×1080（正方形） |
| フレームレート | 60 fps |
| 背景色 | `#0a0a0f`（ほぼ黒） |
| デフォルト実行速度 | `run_time` は各アニメーション指定値に従う |

### カラーパレット

| 名前 | HEX | 用途 |
|------|-----|------|
| `COL_BG` | `#0a0a0f` | 背景 |
| `COL_BLUE` | `#4488ff` | 図形デフォルト（辺の色） |
| `COL_GOLD` | `#ffd700` | 辺の強調・長さを示す数式 |
| `COL_RED` | `#ff4444` | 面積の強調・面積を示す数式 |
| `COL_GREEN` | `#88ff88` | Koch変換の操作中の辺・補助線 |
| `COL_WHITE` | `#ffffff` | 数式が登場するとき（初期） |

### 数式スタイル

- **長さを示す数式**：金色 `COL_GOLD`
- **面積を示す数式**：赤色 `COL_RED`
- **初登場時**：白 `COL_WHITE` でフェードイン後、対応色に切替
- フォント：LaTeX（MathTex）
- 数式は **画面上部中央**（長さ L）と **画面下部中央**（面積 A）に常駐
- 数式の切替はクロスフェード（旧式フェードアウト 0.7s と同時に新式フェードイン）

### 正三角形の定義

画面中央に外接円半径 R の正三角形。頂点を以下とする：

```
top   = (0,  R)          # 上頂点
left  = (-R·√3/2, -R/2) # 左下頂点
right = ( R·√3/2, -R/2) # 右下頂点
```

Manimの座標系（中心原点）に合わせて適宜スケール調整すること。推奨 R = 2.5（Manim単位）。

## Koch曲線の生成アルゴリズム

各辺 [P1, P2] に対して以下を繰り返す（n回でステップn）：

```
M1 = P1 + (P2 - P1) / 3
M2 = P1 + 2*(P2 - P1) / 3
中点 mid = (M1 + M2) / 2
方向ベクトル d = M2 - M1
外向き頂点 peak = mid + rotate_90_ccw(d) * (√3/2)
```

辺 [P1, P2] を [P1, M1, peak, M2, P2] の4線分に置換する。

三角形全体への適用順序：上頂点 → 左下 → 右下 → 上頂点（左回り）。

## フェーズ別アニメーション仕様

---

### Phase 1：三角形と点の登場

**目的**：正三角形と測定開始点を提示する。

**手順**：

1. **三角形フェードイン**（`run_time=1.0s`、`ease=smooth`）
   - 正三角形をデフォルト青 `COL_BLUE`、線幅 2.5pt でフェードイン

2. **2秒静止**

3. **金色の点の登場**（`run_time=0.6s`、`ease=ease_out`）
   - 上頂点 `top` に半径 6px の金色円 `COL_GOLD` をフェードイン

4. **数式登場**：上部に `L = 0`（金色）をフェードイン

**終了状態**：青い正三角形・金の点（上頂点）・上部に `L = 0`

---

### Phase 2：周長の走査

**目的**：点が一周して周長 L₀ を定義する。

**手順**：

1. **点の走査**（`run_time=5.5s`、等速）
   - 金色の点が `top → left → right → top` の順に左回りに移動
   - 通過した辺を金色の軌跡として残す（未通過の辺は青のまま）
   - 上部数式をリアルタイムで更新：`L = 0.00 L₀` → `L = 0.33 L₀` → `L = 0.66 L₀` → `L = 1.00 L₀`

2. **全辺が金色になった状態で** `L = 1.00 L₀` を一時表示（0.7s）

3. **クロスフェードで** `L = L₀` に変更（金色のまま、以後ずっと金色）

**終了状態**：全辺金色・上頂点に金の点・上部に `L = L₀`

---

### Phase 3：面積の導入

**目的**：元の三角形の面積 A₀ を提示し、下部数式エリアに配置する。

**手順**：

1. **内部フラッシュ**（3回点滅、1回 300ms）
   - 三角形内部を薄い赤 `rgba(255,80,80,0.28)` で点灯・消灯

2. **`A = A₀` のアニメーション降下**（`run_time=1.2s`、`ease=ease_out`）
   - 三角形の内部中央にテキスト `A = A₀`（赤色）を出現させ
   - 画面下部の数式エリア（y方向下端付近）まで滑らかに移動
   - 到達直前にフェードアウト

3. **下部数式として** `A = A_0`（赤色）をフェードイン

4. **上部数式** `L = L_0` を（クロスフェードで）再表示

**終了状態**：金の三角形・上部 `L = L₀`（金）・下部 `A = A₀`（赤）

---

### Phase 4：第1回Koch変換（辺の1/3分割と突起生成）

**目的**：各辺を Koch 変換し、周長が 4/3 倍になることを示す。

**変換を3辺に順次適用**（左辺 → 底辺 → 右辺の順）：

#### 左辺のみの処理（サブステップあり）：

**ステップ A**：1/3 分割の可視化（`run_time=0.7s`）
- 左辺（`top → left`）の1/3点・2/3点にティックマーク（垂線、緑）をフェードイン
- 中央1/3の線分を緑 `COL_GREEN` でハイライト
- 上1/3の線分中点付近に `"1/3"` ラベル（緑）を表示
- 0.6s 静止

**ステップ B**：突起の成長（`run_time=0.9s`、`ease=ease_out`）
- 中央の線分 [M1, M2] を取り除き、外向きの等辺三角形の頂点 `peak` に向かって [M1, peak, M2] が成長するアニメーション
- ティックマークを維持して表示
- 成長軌跡は緑で描画

**ステップ C**：`× 4/3` ラベル表示（0.8s 静止）
- 変換した辺の外側（図形と重ならない位置、辺中点から重心と逆方向にオフセット）に `× 4/3` を緑テキストで表示

**ステップ D**：ラベルの処理
- **左辺・底辺**（1・2辺目）：`× 4/3` ラベルをその場でフェードアウト（0.5s）
- **右辺**（3辺目、最終辺）：`× 4/3` ラベルが上部数式エリアへ向かって飛んでいく（`run_time=0.9s`、`ease=ease_in_out`）
  - 移動中に縮小・フェードアウト
  - 到達と同時に上部数式をクロスフェード → `L = 4/3 · L₀`（金）

#### 底辺・右辺：
- ステップ A（分割可視化）はスキップ
- ステップ B → C → D（その場フェードアウト）のみ実行

**終了状態**：Koch ステップ1の雪片型・上部 `L = 4/3 L₀`（金）・下部 `A = A₀`（赤）

---

### Phase 5：第1世代の追加面積

**目的**：3つの突起三角形が各 1/9 A₀ の面積をもつことを示す。

**手順**：

1. **3つの突起三角形を順番にハイライト**（各 0.7s フェードイン + 0.5s 静止）
   - 塗りつぶし `rgba(255,60,60,0.32)`・枠線 赤 1.5pt
   - 各三角形の重心に `1/9 A₀` のラベル（赤）を表示

2. **3つ同時表示**（0.6s 静止）

3. **小さな赤い三角形が下部数式エリアへ飛ぶ**（`run_time=1.1s`、`ease=ease_in_out`）
   - 中央の突起三角形の重心から出発
   - 画面下部中央（数式エリア手前）へ向かって移動しながら縮小（スケール 1.0 → 0.3）
   - 到達直前にフェードアウト（t=0.85 以降）

4. **下部数式をクロスフェード** → `A = A₀ + 3 × 1/9 A₀`（赤）

**終了状態**：Koch ステップ1・突起三角形うっすら赤・下部 `A = A₀ + 3 × 1/9 A₀`

---

### Phase 6：第2回Koch変換

**目的**：辺が再び 4/3 倍になることを示す。

**手順**：

1. **Koch ステップ1 → ステップ2 のモーフィング**（`run_time=2.0s`、`ease=ease_in_out`）
   - 各頂点を線形補間してスムーズに変形
   - 頂点数が異なるため、少ない方をアップサンプリングして一致させる

2. **`× 4/3` ラベルを図形右上外側に表示**（0.7s 静止）

3. **ラベルが上部数式へ飛んでいく**（`run_time=0.9s`）
   - Phase 4 と同様の動作

4. **上部数式クロスフェード** → `L = (4/3)² L₀`（金）

5. **下部数式クロスフェード** → `A = A₀ + 3 × 1/9 A₀`（値変わらず、再表示）

**終了状態**：Koch ステップ2・上部 `L = (4/3)² L₀`・下部 `A = A₀ + 3 × 1/9 A₀`

---

### Phase 7：第2世代の追加面積

**目的**：第2世代の突起が面積式に加算されることを示す。

**手順**：

1. **背景を赤く1回パルス光らせる**（`run_time=0.9s`）
   - `sin(t·π)` のエンベロープで最大透明度 0.22 の赤い塗りつぶし（元の三角形領域全体）
   - 図形は金色で上描き

2. **下部数式クロスフェード** → `A = A₀ + 3·(1/9)A₀ + 3·4·(1/9)²A₀`（赤）

**終了状態**：Koch ステップ2・下部数式更新済み

---

### Phase 8：反復とLの発散

**目的**：Koch 変換を繰り返すと L が∞に発散することを示す。

**手順**：

1. **ステップ2→3→4→5 の連続モーフィング**（各ステップ `run_time=1.5s`、`ease=ease_in_out`）
   - 線幅はステップが増えるにつれて細くする（ステップ5で約0.7pt）
   - 各ステップ完了後に上部数式クロスフェード：
     - `L = (4/3)³ L₀`
     - `L = (4/3)⁴ L₀`
     - `L = (4/3)⁵ L₀`
     - `L = (4/3)⁵ L₀`（ステップ5で5乗）
   - 各ステップ間 0.45s 静止

2. **上部数式クロスフェード** → `L = (4/3)ⁿ L₀ →[n→∞] ∞`（金）

3. **0.9s 静止**

4. **図形内部を赤く1回パルス光らせる**（`run_time=1.3s`）
   - `sin(t·π)` エンベロープ、最大透明度 0.5
   - Koch ステップ5の外形全体（閉じたパスで塗りつぶし）
   - 「長さは∞になったが面積は有限」という対比の予告

**終了状態**：Koch ステップ5の雪片・上部 `L → ∞`・内部赤フラッシュ後消える

---

### Phase 9：面積の収束

**目的**：面積の無限級数が 8/5 A₀ に収束することを示す。

**手順**：下部数式を4段階でクロスフェード。各段階の間に 2.6s 静止。

#### Step 1（2.6s 表示）
```
A = A₀ + 3·(1/9)A₀ + 3·4·(1/9)A₁ + 3·4²·(1/9)A₂ + ···
```

#### Step 2（2.6s 表示）
```
A = A₀ + 1/3 A₀ + 1/3·(4/9)A₀ + 1/3·(4/9)²A₀ + ···
```

#### Step 3（2.6s 表示）
```
A = A₀ ( 1 + 1/3 · Σ[k=0→∞] (4/9)^k )
```

#### Step 4（3.5s 表示・2行レイアウト）
```
A = A₀ ( 1 + 1/3 · Σ[k=0→∞] (4/9)^k )  = A₀(1 + 3/5) = 8/5 A₀
```
- Σの式をそのまま残し、等号でつないで右辺に評価結果を展開
- 数式が長いため2行に折り返す（改行位置：`= A₀(1+3/5) = ...` の手前）
- 赤色のまま

**終了状態**：Koch ステップ5・上部 `L → ∞`・下部 `A = ... = 8/5 A₀`

## 共通アニメーション仕様

### イージング関数

| 名前 | 式 | 用途 |
|------|-----|------|
| `ease_in_out` | `t<0.5 ? 2t² : -1+(4-2t)t` | 大半の移動・変形 |
| `ease_out` | `1-(1-t)²` | フェードイン・突起成長 |
| `ease_linear` | `t` | 周長カウントアップ |

Manim では `rate_func=smooth`（ease_in_out）、`rate_func=rush_into` 等を活用。

### 数式クロスフェード

- 旧数式：`FadeOut(run_time=0.7)`
- 新数式：`FadeIn(run_time=0.7)` を同時実行（`AnimationGroup`）
- Manim では `TransformMatchingTex` または `ReplacementTransform` を適宜使用

### Koch変換のモーフィング

- ステップ n とステップ n+1 の頂点リストを線形補間
- 頂点数が異なる場合は少ない方を等比でアップサンプリングして点数を一致させてから補間
- Manim では `VMobject` の `become()` を `UpdateFromAlphaFunc` と組み合わせる、あるいは `Transform` を使用

### 「飛ぶ」演出

- `× 4/3` テキストや小さな赤三角形が図形から数式エリアへ移動する際：
  - 移動：`ease_in_out` で目標座標へ
  - 縮小：スケール 1.0 → 0.3（`ease_in_out`）
  - フェードアウト：t=0.85 以降に線形でアルファ 1 → 0
  - Manim では `MoveAlongPath` + `ScaleInPlace` + `FadeOut` の組み合わせ、または `UpdateFromAlphaFunc` で手動制御

## Manim 実装上の注意点

1. **Koch曲線の点数**：ステップ5では `4^5 × 3 + 1 = 3073` 点。`VMobject` の `set_points_as_corners()` で構成する。

2. **モーフィングの頂点対応**：`Transform` は点数が一致していないと予期せぬ変形になる。アップサンプリング関数を自前実装して点数を揃えること。

3. **数式の位置**：`MathTex` オブジェクトを `to_edge(UP, buff=0.3)` / `to_edge(DOWN, buff=0.3)` で配置。

4. **クロスフェード**：`FadeOut` と `FadeIn` を `AnimationGroup(lag_ratio=0)` で同時実行。

5. **`× 4/3` の飛行**：`MathTex` を `ValueTracker` + `always_redraw` または `UpdateFromAlphaFunc` で制御。

6. **赤い内部フラッシュ**：Koch曲線の外形パスを閉じて `fill_opacity` を `sin(alpha * PI)` でアニメーション。

7. **フェーズ間のポーズ**：各フェーズ終了後に `self.wait(1.5)` を挿入。
